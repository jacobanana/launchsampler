    Troubleshooting¶

    This guide helps you resolve common issues with LaunchSampler.

    Logging and Diagnostics¶

    Enabling Debug Logging¶

    When troubleshooting issues, enable debug logging to capture detailed information:

    launchsampler --debug

    This creates launchsampler-debug.log in your current directory with detailed diagnostic information.

    For more information about logging options, see the Logging section in Configuration.

    Permission Errors (Linux)¶

    On Linux, you may need to add your user to the audio group:

    sudo usermod -a -G audio $USER

    Then log out and log back in for changes to take effect.

    Audio Issues¶

    Audio Dropouts / Glitches¶

    Symptoms: Crackling, popping, or stuttering audio during playback.

    Solutions:

    1. Increase buffer size:
    launchsampler config --buffer-size 1024

    1. Close other audio applications that might be competing for the audio device

    2. Use a dedicated audio interface if using built-in audio

    3. Check CPU usage - ensure other processes aren't consuming too much CPU

    High Latency¶

    Symptoms: Noticeable delay between pressing a pad and hearing the sample.

    Solutions:

    1. Decrease buffer size (if your system can handle it): 

      launchsampler config --buffer-size 32

    2. Use a better audio API:

    3. Windows: ASIO > WASAPI > DirectSound > MME
    4. macOS: CoreAudio (default)

    5. Linux: ALSA > PulseAudio

    6. Update audio drivers to the latest version

    7. Use a dedicated audio interface with ASIO support (Windows)

    No Audio Output¶

    Symptoms: No sound when triggering samples.

    Solutions:

    1. Verify sample files exist and are valid audio files. When a file can't be loaded, it will be shown as unavailable.

    2. Revert to default audio device: 

      launchsampler config --audio-device default

    3. Check system volume and ensure LaunchSampler isn't muted

    4. Try a different audio device: By default, the audio list command will only show low latency devices, you can see the full list with: 

      launchsampler audio list --all
      Choose a different device ID

    5. Check logs for errors: 

      launchsampler --debug
cat launchsampler-debug.log | grep ERROR

    Wrong Audio Device¶

    Symptoms: Audio plays through wrong output (e.g., speakers instead of headphones).

    Solutions:

    1. List available devices: 

      launchsampler audio list

    2. Set device explicitly in config: 

      launchsampler config --audio-device 3

    3. Restart LaunchSampler after changing config

    Audio Device Unavailable / Changed¶

    Symptoms: Configured audio device is no longer available (e.g., external interface unplugged, device ID changed).

    Automatic Fallback Behavior:

    LaunchSampler automatically handles invalid or unavailable audio devices with the following fallback sequence:

    1. Tries configured device (default_audio_device in config.json)
    2. Falls back to OS default if configured device is invalid
    3. Searches for any low-latency device if OS default doesn't support low-latency APIs
    4. Shows error only if NO valid low-latency devices are found

    Audio Device Already in Use¶

    Symptoms: Application fails to start with error: 

    Error opening OutputStream: Invalid device [PaErrorCode -9996]

    Cause: The audio device is currently in use by another application or instance of LaunchSampler.

    Solutions:

    1. Check for running instances:
    2. Close any other running instances of LaunchSampler

    3. Check task manager/activity monitor for multiple processes

    4. Close other audio applications:

    5. Close DAWs, music players, or other audio software

    6. Check system tray for hidden audio applications

    7. Restart audio services (if needed):

    === "Windows" 

    # Restart Windows Audio service
net stop audiosrv && net start audiosrv

    === "macOS" 

    # Restart CoreAudio
sudo killall coreaudiod

    === "Linux" 

    # Kill processes using audio device
sudo fuser -k /dev/snd/*

# Or restart PulseAudio
pulseaudio -k && pulseaudio --start

    1. Try a different audio device: 

      launchsampler audio list
launchsampler config --audio-device <different-device-id>

    2. Reboot your computer to release all audio device locks

    MIDI Issues¶

    MIDI Device Not Detected¶

    Symptoms: Launchpad not recognized or not showing in device list.

    Solutions:

    1. Check USB connection:
    2. Ensure cable is properly connected
    3. Try a different USB port

    4. Try a different USB cable

    5. List MIDI devices: 

      launchsampler midi list

    6. Install manufacturer drivers:

    7. Download latest drivers from Novation website

    8. Restart computer after installation

    9. Check device permissions (Linux): 

      sudo usermod -a -G audio $USER
# Log out and log back in

    10. Verify device works in other software (e.g., DAW, Launchpad Arcade)

    MIDI Notes Not Triggering Samples¶

    Symptoms: Pressing Launchpad pads doesn't trigger samples in Play Mode.

    Solutions:

    1. Ensure you're in Play Mode:
    2. Press P to switch to Play Mode

    3. Look for "Play Mode" in the status bar

    4. Check that pads have samples assigned:

    5. Switch to Edit Mode (E)

    6. Verify pads show sample information

    7. Monitor MIDI input: 

      launchsampler midi monitor
      Press pads - you should see MIDI messages

    8. Check MIDI routing:

    9. Ensure no other application is capturing MIDI

    10. Close DAWs or other MIDI software

    11. Enable debug logging: 

      launchsampler --debug
      Check for MIDI-related errors

    Launchpad LEDs Not Updating¶

    Symptoms: Launchpad LEDs don't light up or don't reflect pad states.

    Solutions:

    1. Check Launchpad mode:
    2. Some Launchpads have different modes
    3. Ensure it's in "Programmer" or "Live" mode

    4. Refer to your Launchpad's manual

    5. Verify MIDI output connection: 

      launchsampler midi list
      Check that MIDI output port is detected

    6. Reset Launchpad:

    7. Disconnect and reconnect USB

    8. Power cycle if battery-powered

    9. Check logs for LED errors: 

      launchsampler --debug
grep -i "led" launchsampler-debug.log

    Application Issues¶

    Application Won't Start¶

    Symptoms: LaunchSampler crashes or exits immediately on startup.

    Solutions:

    1. Check Python version: 

      python --version  # Should be 3.12 or higher

    2. Enable debug logging: 

      launchsampler --debug 2>&1 | tee startup-error.log

    3. Verify installation: 

      launchsampler --version

    4. Reinstall dependencies: 

      # If using pipx
pipx reinstall launchsampler

# If using uv
cd launchsampler
uv sync --reinstall

    5. Check for conflicting packages: 

      pip list | grep -i midi
pip list | grep -i audio

    6. Try with minimal config: 

      # Backup existing config
mv ~/.launchsampler/config.json ~/.launchsampler/config.json.bak

# Start with defaults
launchsampler

    TUI Display Issues¶

    Symptoms: Garbled text, incorrect colors, or layout problems in the terminal.

    Solutions:

    1. Use a modern terminal:
    2. Windows: Windows Terminal (recommended)
    3. macOS: iTerm2 or Terminal.app

    4. Linux: gnome-terminal, kitty, alacritty

    5. Check terminal size:

    6. Ensure terminal is at least 80x24 characters

    7. Resize terminal window

    8. Set correct locale: 

      export LANG=en_US.UTF-8
export LC_ALL=en_US.UTF-8

    9. Try a different terminal emulator

    10. Check terminal color support: 

      echo $COLORTERM  # Should show "truecolor" or similar

    Application Freezes or Becomes Unresponsive¶

    Symptoms: LaunchSampler stops responding to input.

    Solutions:

    1. Force quit:
    2. Press Ctrl+C in terminal

    3. Or press Ctrl+Q in the TUI

    4. Check CPU usage: 

      # In another terminal
top -p $(pgrep -f launchsampler)

    5. Check for blocking operations: 

      launchsampler --debug
# Look for operations that take too long

    6. Reduce sample count:

    7. Use fewer simultaneous samples

    8. Reduce sample file sizes

    9. Increase buffer size to reduce CPU load: 

      {
  "default_buffer_size": 1024
}

    Samples Not Loading¶

    Symptoms: Error when loading samples from directory or set file.

    Solutions:

    1. Check file permissions: 

      ls -l /path/to/samples/
      Ensure files are readable

    2. Verify file formats:

    3. Supported: WAV, MP3, FLAC, OGG, AIFF

    4. Check file extensions match actual format

    5. Check file paths:

    6. Use absolute paths
    7. Ensure no special characters in paths

    8. Avoid spaces in filenames (or use quotes)

    9. Verify sample directory exists: 

      launchsampler --samples-dir /path/to/samples --debug

    10. Check disk space: 

      df -h

    11. Look for loading errors in logs: 

      grep -i "load" launchsampler-debug.log
grep -i "sample" launchsampler-debug.log

    Configuration Issues¶

    Config File Not Found¶

    Symptoms: LaunchSampler can't find or load config file.

    Solutions:

    1. Check config location:

    === "Linux / macOS" 

    ls -la ~/.launchsampler/config.json

    === "Windows" 

    dir %USERPROFILE%\.launchsampler\config.json

    1. Create config directory: 

      mkdir -p ~/.launchsampler

    2. Let LaunchSampler create default config: 

      launchsampler
# Will auto-create config.json

    3. Manually create config: 

      echo '{"default_buffer_size": 512}' > ~/.launchsampler/config.json

    Invalid Config Values¶

    Symptoms: Config validation errors on startup.

    Solutions:

    1. Validate JSON syntax: 

      python -m json.tool ~/.launchsampler/config.json

    2. Reset to defaults: 

      rm ~/.launchsampler/config.json
launchsampler  # Will create new default config

    3. Check value ranges:

    4. default_buffer_size: 128, 256, 512, 1024, or 2048
    5. default_audio_device: Valid device ID from launchsampler audio list

    6. midi_poll_interval: Any positive float (typically 0.5 to 5.0 seconds)

    7. Review config documentation: See Configuration Guide for all valid options

    Logging Issues¶

    For logging-specific troubleshooting (log files not created, empty logs, etc.), see the Logging Troubleshooting section in the Configuration guide.

    Performance Issues¶

    High CPU Usage¶

    Symptoms: LaunchSampler consumes excessive CPU resources.

    Solutions:

    1. Increase buffer size: 

      {
  "default_buffer_size": 1024
}

    2. Reduce simultaneous voices:

    3. Use one-shot mode instead of loop

    4. Limit number of playing samples

    5. Close other applications

    6. Check for debug logging:

    7. Disable verbose logging in production
    8. Use default logging level (WARNING)

    Slow Sample Loading¶

    Symptoms: Long delay when loading sets or samples.

    Solutions:

    1. Use smaller sample files:
    2. Convert to lower bit depth (16-bit instead of 24-bit)

    3. Use compressed formats (MP3, OGG) for non-critical samples

    4. Reduce sample count:

    5. Split large sets into multiple smaller sets

    6. Load only needed samples

    7. Use SSD instead of HDD for sample storage

    8. Check disk I/O: 

      # Linux
iostat -x 1

# macOS
iostat -w 1

    Platform-Specific Issues¶

    Windows¶

    ASIO4ALL Not Working¶

    Problem: ASIO4ALL driver not recognized.

    Solutions:

    1. Install ASIO4ALL:
    2. Download from http://www.asio4all.org/

    3. Run installer as administrator

    4. Configure ASIO4ALL:

    5. Open ASIO4ALL control panel
    6. Enable your audio device

    7. Set buffer size

    8. Set in LaunchSampler config: 

      {

  "default_audio_device": 0
}

    Permission Errors¶

    Problem: Access denied errors when accessing files or devices.

    Solutions:

    1. Run as administrator (only if necessary)

    2. Check file permissions in Windows Explorer

    3. Use user-writable locations: 

      launchsampler --log-file %USERPROFILE%\Desktop\launchsampler.log

    macOS¶

    CoreAudio Issues¶

    Problem: Audio device not accessible or permission errors.

    Solutions:

    1. Grant microphone permission:
    2. System Preferences Security & Privacy Microphone

    3. Enable for Terminal or your terminal app

    4. Reset Core Audio: 

      sudo killall coreaudiod

    5. Check audio MIDI setup:

    6. Open Audio MIDI Setup.app
    7. Verify device is recognized

    Linux¶

    ALSA Errors¶

    Problem: "Device or resource busy" errors.

    Solutions:

    1. Add user to audio group: 

      sudo usermod -a -G audio $USER
# Log out and log back in

    2. Kill processes using audio device: 

      sudo fuser -k /dev/snd/*

    3. Use PulseAudio plugin: 

      sudo apt-get install pulseaudio-utils

    4. Check ALSA configuration: 

      cat /proc/asound/cards
aplay -l

    PulseAudio Issues¶

    Problem: No audio with PulseAudio.

    Solutions:

    1. Restart PulseAudio: 

      pulseaudio -k
pulseaudio --start

    2. Check PulseAudio devices: 

      pacmd list-sinks

    3. Set default sink: 

      pactl set-default-sink <sink-name>

    Getting Help¶

    If you can't resolve your issue:

    1. Enable debug logging: 

      launchsampler --debug

    2. Gather system information: 

      launchsampler --version
python --version
uname -a  # Linux/macOS

    3. Check existing issues:

    4. GitHub Issues

    5. Create a new issue:

    6. Include debug log (relevant portions)
    7. System information
    8. Steps to reproduce

    9. Expected vs actual behavior

    10. Join the community:

    11. GitHub Discussions
